## üìä System Overview

### Entities
1. **users** - User authentication and profiles
2. **categories** - Product categorization
3. **products** - Inventory items
4. **stock_movements** - Stock transaction history
5. **stock_alerts** - Automated inventory alerts

### Key Features
- **Product Management** - Add, edit, view, and manage inventory items
- **Category Management** - Organize products by categories
- **Stock Movement Tracking** - Record IN/OUT/ADJUSTMENT/RETURN transactions
- **Alert System** - Automated low stock and overstock alerts
- **Activity Logging** - Complete audit trail of all actions
- **Reporting** - Stock movement reports and analytics

### Relationships
- Categories ‚Üí Products (1:M)
- Products ‚Üí Stock Movements (1:M)
- Products ‚Üí Stock Alerts (1:M)
- Users ‚Üí Stock Movements (1:M)

## üîê Business Rules

1. **Stock Validation**
   - Stock cannot go below zero
   - SKU must be unique
   - Products must belong to a category

2. **Alert Generation**
   - Low stock alert when quantity ‚â§ min_quantity
   - Overstock alert when quantity ‚â• max_quantity
   - Alerts are auto-generated by the system

3. **Stock Movements**
   - All movements must be recorded with a user
   - Movements immediately update product quantity
   - Movement types: IN, OUT, ADJUSTMENT, RETURN
